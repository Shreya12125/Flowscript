Complete FlowScript Tutorial

Welcome to FlowScript!

FlowScript is a programming language that uses natural English phrases instead of cryptic symbols. If you can write a sentence, you can write code!

What you'll learn:

  • How to store information in variables
  • How to perform calculations and logic
  • How to make decisions in your programs
  • How to repeat actions efficiently
  • How to organize code into reusable functions
  • How to work with collections of data

Lesson 1: Variables - Storing Information

What is a variable? Think of a variable as a labeled box where you can store a value. You give it a name, and you can put something inside it.

Creating Number Variables

Numbers are used for counting, calculations, ages, prices, scores, and anything involving math.

Syntax:
Create a number called [name] with value [number]

Examples:
Create a number called age with value 25
Create a number called score with value 100
Create a number called price with value 49.99
Create a number called temperature with value -5

Key points:

  • Variable names should be descriptive (use studentAge not x)
  • Numbers can be whole (10) or decimal (10.5)
  • Numbers can be negative (-15)

Creating Text Variables

Text (also called "strings") stores words, sentences, names, messages - anything you want to display or work with as text.

Syntax:
Create a text called [name] with value "[text]"

Examples:
Create a text called name with value "Alice"
Create a text called greeting with value "Hello, World!"
Create a text called city with value "New York"
Create a text called message with value "Welcome to FlowScript"

Key points:

  • Text must be surrounded by quotes: "like this"
  • Text can contain spaces, punctuation, and numbers
  • Text and numbers are different: "25" is text, 25 is a number

Creating Boolean Variables

Booleans store true/false values. They're perfect for yes/no questions, on/off states, or any situation with two possibilities.

Syntax:
Create a boolean called [name] with value [true/false]

Examples:
Create a boolean called isStudent with value true
Create a boolean called hasLicense with value false
Create a boolean called isRaining with value true
Create a boolean called gameOver with value false

Key points:

  • Only two possible values: true or false
  • No quotes around true/false
  • Used for conditions and decision-making

Changing Variable Values

Once created, you can update a variable's value:

Set age to 26
Set name to "Bob"
Set isActive to false
Increase score by 10
Decrease lives by 1

Lesson 2: Output - Showing Information

What is output? Output is how your program communicates with users. It displays information on the screen.

The Say Command

The most common way to show information:

Syntax:
Say [value or text]

Examples:
Say "Hello, World!"
Say age
Say 42
Say isActive

Alternative Output Commands

Display "Welcome!"
Print score
Show message

These all do the same thing - pick the one that reads most naturally to you!

Combining Text with Variables

Use plus to join text and variables together:

Say "Your name is " plus name
Say "You scored " plus score plus " points"
Say "Temperature: " plus temperature plus " degrees"
Display "Welcome, " plus userName plus "!"

Lesson 3: Arithmetic - Doing Math

Why arithmetic? Computers excel at calculations. Let's use that power!

Basic Operations

Addition: plus
Say 5 plus 3 → outputs 8
Say age plus 10

Subtraction: minus
Say 10 minus 4 → outputs 6
Say price minus discount

Multiplication: times
Say 6 times 7 → outputs 42
Say length times width

Division: divided by
Say 20 divided by 4 → outputs 5
Say total divided by count

Remainder: modulo
Say 10 modulo 3 → outputs 1
Say number modulo 2 (useful for checking even/odd)

Storing Calculation Results

Create a number called sum with value x plus y
Create a number called area with value length times width
Create a number called average with value total divided by count
Set result to a times b plus c

Complex Expressions

You can combine multiple operations:

Say 5 plus 3 times 2 → outputs 11
Say (5 plus 3) times 2 → outputs 16
Create a number called total with value price times quantity plus tax

Lesson 4: Comparisons - Testing Conditions

Why comparisons? Programs need to make decisions based on whether something is true or false. Comparisons test relationships between values.

Comparison Operators

Greater than:
age is greater than 18
score is greater than 100

Less than:
temperature is less than 0
speed is less than 60

Greater than or equal to:
age is greater than or equal to 21
score is greater than or equal to passingScore

Less than or equal to:
temperature is less than or equal to 32
attempts is less than or equal to 3

Equals:
name equals "Alice"
answer equals 42

Not equals:
status is not equal to "complete"
count is not equal to 0

Understanding True and False

25 is greater than 18 → true
10 is less than 5 → false
"cat" equals "dog" → false
5 plus 5 equals 10 → true

Lesson 5: Logical Operators - Combining Conditions

Why logical operators? Sometimes you need to check multiple conditions at once, like "Is it both sunny AND warm?" or "Is it either Saturday OR Sunday?"

AND Operator

Both conditions must be true for the result to be true.

age is greater than 18 AND hasLicense equals true
score is greater than 90 AND attendance is greater than 80
isWeekend equals true AND isRaining equals false

Examples:

  • true AND true = true
  • true AND false = false
  • false AND false = false

OR Operator

At least one condition must be true for the result to be true.

day equals "Saturday" OR day equals "Sunday"
score is less than 50 OR attempts is greater than 3
isMember equals true OR hasPass equals true

Examples:

  • true OR true = true
  • true OR false = true
  • false OR false = false

NOT Operator

Reverses a condition - makes true become false and false become true.

NOT isLocked
NOT gameOver
NOT (age is less than 18)

Combining Multiple Operators

age is greater than 18 AND (hasLicense equals true OR hasPermit equals true)
score is greater than 90 OR (score is greater than 80 AND extraCredit equals true)

Lesson 6: Conditionals - Making Decisions

Why conditionals? Real programs need to make choices. "If the user is logged in, show their profile. Otherwise, show the login page."

Simple If Statement

For single-line actions when a condition is true:

Syntax:
If [condition] then [action]

Examples:
If age is greater than 18 then say "Adult"
If score equals 100 then say "Perfect score!"
If temperature is less than 0 then say "Freezing!"

Multi-line If Statement

When you need to do multiple things:

Syntax:
If [condition]:
    [action 1]
    [action 2]

Example:
If score is greater than 90:
    say "Excellent work!"
    increase bonus by 10
    set grade to "A"

Important: Indent the lines inside the If block with spaces!

If-Else Statement

Do one thing if true, something else if false:

If age is greater than or equal to 18:
    say "You can vote"
Else:
    say "You cannot vote yet"

If-Else If-Else Statement

Check multiple conditions in sequence:

If score is greater than or equal to 90:
    say "Grade: A"
Else if score is greater than or equal to 80:
    say "Grade: B"
Else if score is greater than or equal to 70:
    say "Grade: C"
Else if score is greater than or equal to 60:
    say "Grade: D"
Else:
    say "Grade: F"

Nested Conditionals

Put if statements inside other if statements:

If hasAccount equals true:
    If password equals correctPassword:
        say "Welcome back!"
    Else:
        say "Wrong password"
Else:
    say "Please create an account"

Lesson 7: Loops - Repeating Actions

Why loops? Imagine needing to print "Hello" 100 times. Without loops, you'd write 100 lines. With loops, you write 2 lines!

Repeat Loop (Fixed Number of Times)

When you know exactly how many times to repeat:

Single-line syntax:
Repeat 5 times: say "Hello"

Multi-line syntax:
Repeat 10 times:
    say "Counting..."
    say "Still counting..."

Real examples:
Repeat 3 times: say "Hip hip hooray!"
Repeat 5 times:
    increase score by 10
    say "Score: " plus score

While Loop (Condition-Based)

Repeat as long as a condition is true. Great when you don't know how many iterations you need:

Syntax:
While [condition]:
    [actions]

Example - Count from 1 to 10:
Create a number called counter with value 1
While counter is less than or equal to 10:
    say counter
    increase counter by 1

Example - Keep guessing:
Create a number called guess with value 0
Create a number called target with value 7
While guess is not equal to target:
    say "Guess again!"
    increase guess by 1

Warning: Make sure your condition eventually becomes false, or you'll create an infinite loop!

For Each Loop (Working with Collections)

Process every item in a list or array:

Syntax:
For each [item] in [array]:
    [actions using item]

Example:
Create an array called names with values ["Alice", "Bob", "Charlie"]
For each name in names:
    say "Hello, " plus name

Nested Loops

Loops inside loops - powerful but use carefully:

Multiplication table:
Create a number called i with value 1
While i is less than or equal to 3:
    Create a number called j with value 1
    While j is less than or equal to 3:
        say i plus " times " plus j plus " = " plus (i times j)
        increase j by 1
    increase i by 1

Practice Exercise 7

Countdown from 10 to 1:
Create a number called countdown with value 10
While countdown is greater than 0:
    say countdown
    decrease countdown by 1
Say "Blast off!"

Lesson 8: Functions - Organizing Your Code

Why functions? Functions let you write code once and use it many times. They help organize complex programs into manageable pieces.

Basic Function Definition

Syntax:
Define function [name] with parameter [param]:
    [function body]

Simple example:
Define function greet with parameter name:
    say "Hello, " plus name plus "!"

Calling the function:
Call greet with "Alice"
Call greet with "Bob"
Call greet with "Charlie"

Functions with Multiple Parameters

Define function add with parameters a and b:
    Create a number called sum with value a plus b
    say sum

Call add with 5 and 3
Call add with 10 and 20

Functions That Return Values

Return lets a function send a value back to use elsewhere:

Define function multiply with parameters x and y:
    return x times y

Create a number called result with value multiply(4, 5)
Say "4 times 5 equals " plus result

Functions with Logic

Check if number is even:
Define function isEven with parameter num:
    If num modulo 2 equals 0:
        return true
    Else:
        return false

Create a boolean called result with value isEven(10)
Say "Is 10 even? " plus result

Recursive Functions

A function that calls itself - powerful for certain problems:

Calculate factorial (5! = 5 × 4 × 3 × 2 × 1):
Define function factorial with parameter n:
    If n is less than or equal to 1:
        return 1
    Else:
        return n times factorial(n minus 1)

Say factorial(5) → outputs 120

Lesson 9: Arrays - Working with Collections

Why arrays? Often you need to store multiple related values. Instead of creating score1, score2, score3... you can use one array to hold all scores!

Creating Arrays

Syntax:
Create an array called [name] with values [list]

Number arrays:
Create an array called scores with values [85, 90, 78, 92, 88]
Create an array called prices with values [9.99, 19.99, 14.99]

Text arrays:
Create an array called names with values ["Alice", "Bob", "Charlie"]
Create an array called cities with values ["New York", "London", "Tokyo"]

Empty array:
Create an array called myList with values []

Accessing Array Elements

Arrays use index numbers starting at 0:

Create an array called fruits with values ["apple", "banana", "orange"]
Say fruits[0] → outputs "apple" (first item)
Say fruits[1] → outputs "banana" (second item)
Say fruits[2] → outputs "orange" (third item)

Modifying Arrays

Change an element:
Set scores[0] to 95
Set names[1] to "Robert"

Add to end of array:
Add 100 to scores
Add "David" to names

Get array length:
Say length of scores
Create a number called count with value length of names

Looping Through Arrays

Using For Each:
Create an array called numbers with values [1, 2, 3, 4, 5]
For each num in numbers:
    say num times 2

Using While with index:
Create a number called i with value 0
While i is less than length of numbers:
    say numbers[i]
    increase i by 1

Array Operations

Sum all elements:
Create an array called nums with values [10, 20, 30, 40]
Create a number called total with value 0
For each n in nums:
    Set total to total plus n
Say "Total: " plus total

Find maximum:
Create a number called max with value nums[0]
For each n in nums:
    If n is greater than max:
        Set max to n
Say "Maximum: " plus max

Lesson 10: Comments - Documenting Your Code

Why comments? Comments explain what your code does. They're notes for yourself and others who read your code.

Single-Line Comments

# This is a comment - it's ignored by the program
Create a number called age with value 25 # Store user's age
Say "Hello" # Greet the user

When to Use Comments

  • Explain complex logic: "# Check if number is prime"
  • Describe purpose: "# Calculate monthly payment"
  • Mark sections: "# --- User Input Section ---"
  • Leave notes: "# TODO: Add error checking"
  • Disable code temporarily: "# Say debugInfo"
Example with good comments:
# Calculate circle area
# Formula: area = π × radius²
Create a number called pi with value 3.14159
Create a number called radius with value 5
Create a number called area with value pi times radius times radius
Say "Circle area: " plus area

Comment Best Practices

  • Write clear, concise comments
  • Explain WHY, not just WHAT
  • Update comments when you update code
  • Don't over-comment obvious code

Advanced Topics

Combining Everything

Real programs combine all these concepts. Here's a complete example:

# Student grade management system
Create an array called students with values ["Alice", "Bob", "Charlie"]
Create an array called grades with values [85, 92, 78]

# Function to determine letter grade
Define function getLetterGrade with parameter score:
    If score is greater than or equal to 90:
        return "A"
    Else if score is greater than or equal to 80:
        return "B"
    Else if score is greater than or equal to 70:
        return "C"
    Else:
        return "F"

# Display all student grades
Create a number called i with value 0
While i is less than length of students:
    Create a text called name with value students[i]
    Create a number called score with value grades[i]
    Create a text called letter with value getLetterGrade(score)
    Say name plus ": " plus score plus " (" plus letter plus ")"
    increase i by 1

Problem-Solving Approach

  1. Understand the problem: What do you need to accomplish?
  2. Break it down: What smaller steps are needed?
  3. Choose the right tools: Do you need loops? Functions? Arrays?
  4. Write pseudocode: Plan in plain language first
  5. Code incrementally: Write small pieces, test often
  6. Debug: Use the Step feature to find issues
  7. Refine: Make your code clearer and more efficient

Using the FlowScript IDE

Control Buttons

  • Run: Execute your entire program
  • Step: Run one line at a time (debugging)
  • Pause: Pause execution during step mode
  • Stop: Halt the program
  • Clear: Reset the editor
  • Examples: Load pre-written programs
  • Format: Auto-indent your code

Output Tabs

  • Output: See what your program displays
  • Flowchart: Visual diagram of program flow
  • Translations: See equivalent Python, Java, JavaScript, C++ code
  • Syntax Tree: How FlowScript parses your code
  • Variables: Monitor all variable values
  • Tokens: See how code is broken into tokens

Tips for Success

  • Start with simple programs and gradually add complexity
  • Use the Examples to learn new techniques
  • Check the Flowchart to visualize your logic
  • Use Step mode to understand how your code executes
  • Read error messages carefully - they tell you what's wrong
  • Adjust font size for comfortable coding
  • Save your favorite programs by copying them
